Kebutuhan non-fungsional merupakan spesifikasi yang mendefinisikan atribut kualitas, batasan, serta karakteristik teknis dari sistem yang tidak berkaitan langsung dengan fungsi-fungsi spesifik. Kebutuhan ini mencakup teknologi pengembangan yang digunakan, lingkungan perangkat lunak, dan spesifikasi perangkat keras yang diperlukan guna mendukung operasional sistem secara keseluruhan.

a.	Teknologi Pengembangan Sistem
    Sistem Informasi Penjualan ERNA Thrifting dibangun menggunakan framework Laravel 11 berbasis bahasa pemrograman PHP untuk mengembangkan fungsi back-end. Laravel menerapkan pola arsitektur Model-View-Controller (MVC) yang menangani logika bisnis, routing URL, serta interaksi dengan basis data. Pada sisi front-end, implementasi tampilan dilakukan menggunakan Blade Template Engine yang terintegrasi dengan Laravel bersama dengan teknologi pendukung HTML dan CSS. Framework CSS kustom digunakan untuk perancangan antarmuka yang responsif dengan dukungan Font Awesome 6 untuk ikonografi serta Google Fonts (Playfair Display dan Montserrat) untuk tipografi. JavaScript digunakan untuk penanganan permintaan AJAX, manipulasi DOM, serta integrasi dengan pustaka pendukung seperti Chart.js untuk visualisasi grafik, AOS (Animate on Scroll) untuk animasi, SweetAlert2 untuk notifikasi interaktif, dan Select2 untuk komponen pencarian. Untuk manajemen data, sistem menggunakan basis data relasional MySQL yang terintegrasi dengan Laravel melalui Eloquent Object Relational Mapping (ORM).

b.	Kebutuhan Perangkat Lunak
    Perangkat lunak merupakan komponen fundamental yang mendukung operasional sistem secara menyeluruh. Kebutuhan perangkat lunak mencakup lingkungan pengembangan yang mendukung PHP versi 8.2 ke atas, Laravel 11 sebagai framework utama, serta MySQL sebagai DBMS. Sistem juga memerlukan Composer sebagai dependency manager PHP, Node.js dan Vite sebagai module bundler untuk aset front-end, serta web server (Apache atau Nginx) untuk menjalankan aplikasi.
